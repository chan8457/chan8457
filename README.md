서울시 공공자전거 **따릉이의 재정 적자 문제**를 개선하기 위해, 요금 인상 대신 **LED 패널 광고 도입 방안**을 제안한 프로젝트입니다.

### 2. 채용 플랫폼 지원 퍼널 진단 및 단계별 개선 제안
이 프로젝트는 **채용 플랫폼의 전환율 하락 원인 진단 및 개선 제안**을 목표로 진행되었습니다.  
2022~2023년 MAU 하락이 두드러진 시점의 데이터를 분석하여,  
‘방문 → 공고 열람 → 이력서 작성 → 지원 완료’로 이어지는 **4단계 퍼널 전환율을 추적**했습니다.  

- **방문 → 공고 열람**: 경력 필터 중요도는 높지만 사용량 급감 → 세분화된 경력 구간 기반 **A/B TEST 제안**  
- **공고 열람 → 이력서 작성**: 생소한 스타트업 공고 다수로 신뢰 신호 부족 → RFM 기반 기업 세그먼트 후 **TOP 50/100 태그 노출 전략**  
- **이력서 작성 → 지원 완료**: 가장 큰 이탈 구간으로 확인 → **지원 현황 요약 박스 A/B TEST 제안**  

데이터 기반으로 각 퍼널 단계의 병목 원인을 도출하고, 구체적인 실험 가설로 개선 방향을 제시했습니다.

### 4. 서비스 종료된 10대 익명 투표 앱 데이터 분석
마지막 프로젝트는 **서비스 종료된 10대 익명 투표 앱의 로그 데이터**를 활용한 고급 분석 과제입니다.  
‘신입 데이터 분석가로서 신규 서비스 재출시 시나리오를 설계한다’는 가정 하에 진행했습니다.  

핵심 분석 축은 두 가지입니다:  
- **정착 실패 분석 (첫 24시간 내 행동 패턴)**  
- **이탈 예측 (7일 미복귀 예측 모델)**  

서비스는 피크 대비 DAU가 **97.7% 급락(316,872 → 7,429)** 했고,  
가입자 중 70%는 핵심 기능(질문/투표)을 경험하지 못했으며, 첫 주 이탈률은 44%에 달했습니다.  

이를 해결하기 위해  
- 첫 세션 내 ‘첫 투표 경험’ 유도 플로우 설계  
- 중심학생 네트워크(Influencer Node) 유지 전략  
- LightGBM 기반 **조기 이탈 경보 시스템**을 구축했습니다.  

또한 Airflow → GCS → BigQuery(Bronze/Silver/Gold) → BQML 학습 파이프라인을 구성하고,  
**대시보드 및 Discord 알림 자동화 시스템**까지 구현했습니다.
